  • Learning Discussion
      Taking greater cyber security to protect medical devices is more important now than ever. Medical training devices are becoming more advanced and increasingly dependent on technology which makes them vulnerable to cybersecurity threats and attacks. A team of researchers at the University of South Alabama investigated potential breaches of medical devices used in medical training using iStan medical mannequin as its prime target. A team of computing students carried out these tests and were able to successfully hack into the medical mannequin and eventually kill him. The network security solution and the network protocols were identified as the vulnerable components and through this, the team was able to carry out brute force attacks as well as denial of service attacks Glisson et al (2015).

      Firstly, the network security solution was breached using an open-source software known as Reaver to instigate a brute force attack against the Wi-Fi Protected Setup register PIN numbers. Two steps were used for the brute force attack using the open-source software Reaver which is included with Backtrack 5 from a live CD which took slightly over seven hours while the same brute force attack was tried on the mannequin using the virtual machine which cracked the passphrase in two hours thirty-eight minutes and forty-eight seconds Glisson et al (2015). Mitigation steps could be taken to forestall this attack by either updating the firmware of the access point with firmware that addresses this vulnerability or disabling the WPS which is what the reaver tool used to breach the network security solution (‘Wi-Fi Protected Setup (WPS) Vulnerable to Brute-Force Attack’, 2012).

      Secondly, the network protocols used by iStan consist of TCP and 802.11 wireless transmissions between the controlling laptop and the iStan's internal access point. The network protocol was compromised using HPING3, a testing tool available in windows and Linux to instigate a DoS attack. The DoS attack overwhelmed the network between the mannequin and the controlling computer, and the interface displayed a connection error message during the DOS attack which made the program unable to send any setting changes to the iStan medical mannequin Glisson et al (2015). These DOS attacks can be mitigated by using any firewalls and intrusion detection systems that act as traffic scanning barriers between networks.

    3. Peer Response - Osarodion Samuel Tolofari


      Hi Abidemi,
      By disabling WPS, the mannequin can't easily connect to the network, making it more vulnerable. The WiFi Protected Setup (WPS) is a wireless network security standard that makes connections between a router and wireless devices (mannequins) faster and easier. Wireless Protected Setup is a key exchange mechanism rather than a security mechanism. It facilitates connections between devices and access points and is one of the vulnerable points of a network.
      WPS works with WPA/WPA2/PSK and allows devices to establish connections to WLANs. Utilizing the following methods
      PIN: Entering a PIN on the device.
      PUSH-BUTTON: A physical or virtual push-button should be pressed on both the access point and the client device during the connection phase.
      NFC: To use Near Field Communication, the client must bring their device close to the access point.
      USB drive: Necessary connection information exchange between an access point and a device is done using a USB drive.
      The short length of WPS PINs and the fact that both PIN parts are validated separately means that an online brute force attack on the PIN can be made within several hours, allowing an attacker to access a WLAN (mannequin device)
      Wireless LAN clients can authenticate their users and ensure user confidentiality through WPA or WPA2. WPA is vulnerable to the offline dictionary attack since the information required to create and verify a session key is broadcasted (Noureddine Boudriga, 2019, p.270) Using WPA2/enterprise security and mutual client and server authentication provides the most secure and protected wireless network compared to the WPA and WEP.

    The ‘Compromising a Medical Mannequin’ research paper by Glisson et al’s (2015), explored the impact of security breaches on medical mannequin’s within medical training facilities. Medical training devices have become increasingly reliant on technology, leading to a growing need to access and mitigate possible security vulnerabilities. The main risks discussed in the Compromising a Medical Mannequin paper (Glisson et al, 2015) are:


    The main risks discussed in the Compromising a Medical Mannequin paper (Glisson et al, 2015) are: - Brute force attacks against 802.11 wireless transmissions, specifically the Wi-Fi Protected Setup (WPS) feature.
    - DOS attacks on the TPS network protocol used by the mannequin to communicate with the machine controlling it.
    When assessing these risks, it is useful to assign each with the risk rating which helps when deciding on the risk response. For this purpose the DREAD risk calculation method (Meier, 2003) can be used. DREAD scores each risk from 0 (no risk) to 3 (high risk) in the following 5 categories:
    - Damage potential – the amount of damage the exploited risk could provide.
    - Reproducibility – the easiness of attack reproduction.
    - Exploitability – how easy to launch the attack?
    - Affected users – the quantity/percentage/bracket of affected users.
    - Discoverability – how easy is it to find the vulnerability?

    Using the above DREAD definition, and the scale 0 to 3, the two risks identified above were scored as follows:

    DREAD category Brute force attack DOS attack
    Damage potential 3(attacker could gain access to the point of login details, thus getting access to the whole network) 1(the mannequin would be taken offline impacting only the small group of people)
    Reproducibility 3(the attack can be reproduced at anytime) (the attack can be reproduced at anytime)
    Exploitability 3(the attack can be carried by anyone with basic computer understanding) 3(the attack can be carried out by anyone with basic computer understanding)
    Affected users 3(after gaining access point security details, the follow up attack could impact the whole network) 1(DOS would impact users of the particular mannequin only)
    Discoverability 3(WPS vulnerability has been known for a long time, the step by step guides are available) 3(The TPS network protocol vulnerabilities are well known)

    The resulting risk total for a brute force attack is 15 (high risk) and 11 (Medium risk). Thus, the brute force attack on a medical mannequin would have a larger impact on the medical training facility than the DOS attack. Please note, that the ‘Affected users’ DREAD category was considered to also include other users within the medical training facility network and not only those directly working with the training mannequin.

    Formative Activities: Summary Post for Collaborative learning Discussion 1 which was centered on the paper Compromising a Medical Mannequin

    Summary post
    Healthcare devices represent one of the fastest-growing sectors of the IoT market with an estimated value of $176 billion by 2026 (Fortune Business Insights, 2020). However, despite the numerous benefits IoT devices offer in the healthcare sector, they have also introduced risks to organizations. A team of students hacked into the iStan Mannequin by breaching the security solution using an open-source brute force attack and attacking the network protocol through a denial-of-service attack within a few hours Glisson et al. (2015).

    The wireless access point was using the WPS and WPA PSK which was easily cracked by reaver. To mitigate this risk, the easiest option was to disable WPS manually or implement a lock-down period to avoid brute force attacks (Swan, N.D). The implementation of WPA2-based encryption and authentication can also be used. Necat (2022) raised the concern about the IoT devices being limited in resources like memory which might make them sometimes unable to use robust encryption, Elliptic Curve Cryptography (ECC) could be used for these IoT devices.

    The network protocol was breached by a DOS attack against the iStan internal access point using the HPING3 tool. Becan (2022) suggested the use of a strong firewall as the first line of defense against DoS attacks. The wireless intrusion detection systems (NIDS) can also be used to constantly scan the network traffic for suspicious events. Other options include the use of VLANs as suggested by Ashmore (2022).

    With the increasing number of healthcare-industry security breaches, the issue of security concerning IoT devices will come into the limelight (Mathews, 2016). 2022 State of Healthcare IoT Device Security report from Cynerio states that "More than half of connected medical and IoT devices in hospitals have a known critical vulnerability" (Cynerio, 2022). There is therefore an urgent need to address these security challenges to make the most of IoT for healthcare.

    During the Unit 3 we have been asked to perform quick individual scans on the website assigned to our team (Group 1): https://loadedwithstuff.co.uk/
    Our team used a combination of traceroute, tracert, and tracepath to record the number of hops to our destination. The number varied between 10 and 15. The hop with the highest latency, presented itself between the middle and the end of the traceroute, i.e. between hops 7 and 11 respectively. No lost packets were observed.
    The differences in the hop numbers can be explained by our geographical locations (Fei et al, 1998) and our internet providers; our team members have conducted these scans from the UK, Canada, Qatar and Nigeria. The latency differences will be due to the differences between our bandwidth.
    Next we were asked to find the mail and name servers for the domain. We used a combination of nslookup and web-based tools to identify the name servers below. This indicated to us that the site is being hosted by A2 Hosting
    • ns1.a2hosting.com
    • ns2.a2hosting.com
    • ns3.a2hosting.com
    • ns4.a2hosting.com
      • The MX record for the assigned site was confirmed by everyone to be mail.loadedwithstuff.co.uk.
    Next we were asked to provide the registered contact for the domain. Again we used a combination of the whois command and various web-based resources. We were able to confirm that the registrar for the site is:
    • eNom LLC [Tag = ENOM]
    • URL: http://www.enom.com
    While contact information for domains used to be public information, this was changed as a result of the introduction of the GDPR when the Internet Corporation for Assigned Names and Numbers (ICANN) agreed to start treating this data as personal information that needs to be kept private (Goyal, 2018).
    The final part of the scan was to identify the physical location of the server hosting the site. Using a variety of web-based IP address geolocation services, we initially suspected that the server was hosted in the USA. However, IP address geolocation is not always accurate, especially when considering addresses assigned to Internet companies, which are free to assign their addresses without regard to geography (IPLocation, 2016). With this in mind, we looked more closely at the network hops identified in our trace routes. These routes all seemed to point to Amsterdam as the correct location. One of our group resides in North America, which made their route particularly interesting; if the site was in the USA then their route should be quite short. Instead, the route showed a trans-Atlantic hop from Montreal to Liverpool which would not have been necessary if the site was in the USA (Pepelnjak, 2019). Based on all of the evidence available, we concluded that the hosting server is most likely located in Amsterdam.

  • Initial Post - Compromise of the Adobe network.

    The case study used refers to the compromise of the adobe network. The compromise of adobe network was announced by adobe on October 3rd, 2013 as a data breach (DesJardins, 2014). The hackers gained access to sensitive personal information like customers' email addresses, payment data, customer names, password hints, and physical addresses. This affected about 150million accounts which comprised active and inactive users (DesJardins, 2014). The active accounts were about 38million with about 3 million customers’ credit and debit card information exposed. This also included the theft of some Adobe products like adobe reader and source code. An old database server which was due to be decommissioned was the entry point for the attackers (Duckett, 2015).
    Although the adobe data breach occurred before GDPR came into effect May 25, 2018 but nevertheless GDPR Art 33 and Art 34 would have addressed this data breach. Art 33 specifically states that the controller should report to the supervisory authority within 72 hours of being aware of it. Art 34 also states that the data subject(customers)must be notified without undue delay anytime personal data breach occurs (Intersoft Consulting, N.D). Adobe was aware of the breach from about September 17th, 2013 but there was no official announcement until October 3rd 2013, which is more than two weeks after being aware of the incident which would have put them at a high risk of penalties or fines (DesJardins, 2014).
    The hackers were able to exploit vulnerabilities on the public facing server like the fact that although the customers passwords were encrypted but the password hints were not and a lot of the passwords were easy to guess through this unencrypted password hints. Complex passwords were not enforced so a lot of customers used simple passwords like 123456, password, adobe123 etc. (DesJardins, 2014).
    As an Information Security manager, I would have implemented the below as a means of mitigating against this data breach
    1. Passwords hints should always be encrypted if needed at all. Complex passwords should also be enforced so that passwords will be difficult to guess.
    2. Multi Factor authentication should be enforced for all customers and employees.
    3. The server in question might not have the latest SW updates since it was due to be decommissioned. All prospective decommissioned servers should be taken off the network and internet to prevent hackers from accessing the network though these old unused servers.

    • Debate Opening Argument 5: Seamless Consumer Mobility
      Mobile devices in the TCP/IP world require an ip address to communicate with other devices which means that for mobile devices, when the physical location is changed, it needs to acquire a new IP address for communication to be feasible. (Saxena et al, 2016)
      The current internet architecture(TCP/IP) has a long standing challenge with host mobility when wireless networks integrate the internet into their architectures. The current internet architecture has a huge proportion of traffic being attributed to mobile devices using mobile data especially for video streaming services. (Tyson et al, 2012)
      Future internet architectures like content-centric architectures (NDN, CCN) propose a key paradigm shift for consumer mobility which involves replacing the current Internet architecture’s existing host-centric paradigm with an information(content)-centric networking paradigm which supports data access by contents rather than IP addresses. (Tyson et al, 2012)
      This added benefit enables mobile users to have better data access since they do not have to keep acquiring IP addresses every time their location changes which will allow them continue their communication during unavailability of published content or topological change.
